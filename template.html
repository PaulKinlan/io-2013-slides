Viewport

The most critical element for mobile TODAY!

Device width:

<meta name="viewport" content="width=device-width" />

Initial scale:

<meta name="viewport" content="initial-scale=1.0" />
<meta name="viewport"
  content="width=device-width, initial-scale=1.0" />

Device Adaptation

A saner solution is on it's way!

@viewport {
  width: device-width;
  orientation: portrait;
}

Greater flexibility when used with Media Queries:

@media screen and (min-device-width: 800px) { 
  @viewport {
    width: device-width;
    user-zoom: fixed;
    zoom: 1.0;
  }
}

Media Queries

Adaptive to screen size

@media screen and (min-device-width: 800px) { ... }
@media screen and (device-aspect-ratio: 16/9) { ... }
@media screen and (min-device-pixel-ratio: 3/2) { ... }
@media screen and (resolution: 2dppx) { ... }
  • Logically group your styling rules.
  • Dynamically responsive.

Orientation

Mobile devices can usually be rotated

@media screen and (orientation: portrait) {}
@media screen and (orientation: landscape) {}

Viewport units

Sizing relatively to the viewport

#visible-square {
  width: 100vmin;
  height: 100vmin;
}
  • Four new units: vw(idth), vh(eight), vmin and vmax.
  • Relative to the size of the viewport

Flexible Box Module

Adaptive layouts

element {
  display: flex;
  flex-direction: row | row-reverse | column |
      column-reverse;
}
  • Adaptive to display size.

Spec: http://dev.w3.org/csswg/css-flexbox/

position: sticky

For The Win!

  • Initially a relatively positioned element.
  • Becomes like position: fixed when parent partially out of view.
  • Hidden when parent is completely out of view.
.sticky-element {
  position: sticky;
  top: 10px;
}

Storage APIs

There's localStorage..

localStorage["test"] = 1234;
var test = localStorage["test"];

..and sessionStorage!

sessionStorage["test"] = 1234;
var test = sessionStorage["test"];
  • sessionStorage only works for the current session, localStorage is persistent
  • Both are key-value data stores

WebSQL

Relational databases for JavaScript

var database = openDatabase('animals', ...);
database.transaction(function (tx) {
  tx.executeSql('INSERT INTO animals VALUES ("cat", "awwww ❤");');
}, function(error) {
  // Couldn't store this lovely cat :-(
));
  • Deprecated, but still has fair support.

IndexedDB

A new database for the Web?

var database = indexedDB.open('animals', 1);
database.onsuccess = function (event) {
  var request = database.transaction("[animals]")
                        .objectStore("animals")
                        .get("cat");
  request.onsuccess = function(event) {
    request.result.comment; // awwww ❤
  };
};
  • IndexedDB is gaining support.

Camera - Media Capture

Take photos

cam.onchange = function(event) {
  // An image has been captured.
  if(event.target.files.length == 1 &&
     event.target.files[0].type.indexOf("image/") == 0) {
     // We have an image, most likely from a camera
  }
}

getUserMedia

Inline camera and mic access

navigator.getUserMedia({audio:true, video: true}, function(stream) {
  // An image has been captured.

  video.play();
};
